Reference Sheet

Key formulas & definitions

The most important formulas from all chapters. Memorise these before touching past papers.

Chapter 2 — Cellular System Formulas
  • Co-channel reuse ratio: Q = D/R = √(3N) where N = i² + ij + j²
  • Cluster size: N = i² + ij + j² (valid values: 1, 3, 4, 7, 9, 12...)
  • SIR (omni): S/I = (√3N)ⁿ / 6 where n = path loss exponent
  • Traffic intensity per user: A_u = λ · H (call arrival rate × avg call duration in Erlangs)
  • Total offered traffic: A = A_u × U (U = number of users)
  • Erlang B (GOS): Use the Erlang B table provided with the exam (always attached). For N channels, find A such that B% blocking is satisfied.
  • Cell area (hexagonal): A_c = 2.598 R² where R is cell radius
  • Min distance between co-channel cells: D = R√(3N)
  • Market penetration: (Total supportable users / Area population) × 100%
Chapter 3 — Propagation Model Formulas
  • Free-space path loss: Pr = Pt · Gt · Gr · (λ/4πd)²
  • Free-space PL in dB: PL(dB) = 20log(d) + 20log(fc) + 20log(4π/c)
  • Far-field distance: d_f = 2D²/λ (D = max antenna dimension)
  • Two-ray model: Pr = Pt · Gt · Gr · (ht · hr)² / d⁴ (for d >> ht·hr)
  • Hata model (Urban, 150–1500 MHz): L₅₀(dB) = 69.55 + 26.16·log(fc) − 13.82·log(hte) − a(hre) + (44.9 − 6.55·log(hte))·log(d)
  • Correction factor a(hre) — medium city: a(hre) = (1.1·logfc − 0.7)·hre − (1.56·logfc − 0.8) dB
  • Correction factor a(hre) — large city (fc ≥ 300 MHz): a(hre) = 3.2(log(11.75·hre))² − 4.97 dB
  • Suburban Hata: L₅₀(suburban) = L₅₀(urban) − 2[log(fc/28)]² − 5.4
  • RMS delay spread: σ_τ = √(Στ²P(τ) / ΣP(τ) − τ̄²)
  • Coherence bandwidth (90%): Bc ≈ 1/(50·σ_τ)
  • Coherence bandwidth (50%): Bc ≈ 1/(5·σ_τ)
  • Coherence time: Tc ≈ 0.423/fm where fm = v·fc/c (max Doppler freq)
  • Doppler shift: fd = (v/λ)·cosθ = v·fc·cosθ/c
Chapter 7 — Multiple Access Formulas
  • FDMA channels: N = (B_total − 2·B_guard) / B_channel
  • TDMA frame efficiency: η = (1 − b_overhead/b_frame) × 100%
  • GSM TDMA frame efficiency example: Total frame = 156.25 bits, overhead = 6+8.25+26 = 40.25 bits → η ≈ 74%
  • CDMA spreading: Chip rate = Rb × processing gain. User signal XOR with code. Despreading = correlate with same code.
  • Processing gain: PG = BW_spread / BW_signal = Rc/Rb
Chapter 4 — Modulation Key Points
  • QPSK: 2 bits/symbol, 4 phase states (0°, 90°, 180°, 270°). I and Q channels modulated simultaneously.
  • OQPSK: Like QPSK but Q channel delayed by T/2. Avoids 180° phase transitions → reduces envelope fluctuation.
  • π/4-QPSK: Rotates constellation by 45° each symbol. Max phase shift = 135°. Used in IS-136 TDMA.
  • MSK: Continuous phase FSK. Modulation index h=0.5. Spectral efficiency better than BFSK.
  • GMSK: MSK with Gaussian pre-filter. Tighter spectrum. Used in GSM (BT=0.3).
  • OFDM key principle: Divides wideband channel into N narrowband orthogonal subcarriers. Cyclic prefix prevents ISI. IFFT at transmitter, FFT at receiver.
  • OFDM advantage: Resistant to multipath. Spectral efficient. Simple equalization in frequency domain.
Chapter 8 — GSM Channels Quick Reference
  • Traffic Channel (TCH): Carries voice/data for actual calls (Full-rate: TCH/F, Half-rate: TCH/H)
  • BCCH (Broadcast Control Ch.): Downlink only. Broadcasts cell system info (frequencies, timing, identity). Always transmitted at constant power.
  • RACH (Random Access Ch.): Uplink only. MS requests connection to network.
  • PCH (Paging Ch.): Downlink only. Network pages MS for incoming calls.
  • AGCH (Access Grant Ch.): Downlink. Assigns SDCCH in response to RACH.
  • SDCCH (Stand-alone Dedicated CCH): Two-way. Used for call setup before TCH assigned.
  • SACCH (Slow ACCH): Two-way. Carries measurement reports and control. Associated with TCH.
  • FACCH (Fast ACCH): Steals frames from TCH. Used for handoff signalling.
  • GSM Frame structure: 1 frame = 8 timeslots. 26-frame multiframe (voice) or 51-frame multiframe (control). 1 superframe = 1326 frames.
  • GSM Architecture blocks: MS → Um → BTS → Abis → BSC → A → MSC → PSTN. HLR, VLR, AuC, EIR are in NSS.

FDMA Channel Count (Guaranteed Numerical)

N = (B_total − 2×B_guard) / B_channel

Example: (12.8 MHz − 2×10 kHz) / 30 kHz
= (12,800,000 − 20,000) / 30,000
= 12,780,000 / 30,000 = 426 channels
This exact problem appears in 2079, 2078, 2077, 2075 and earlier papers. Memorise the substitution.

Hata Model — Urban Path Loss

L₅₀ = 69.55 + 26.16·log(fc)
− 13.82·log(hte) − a(hre)
+ (44.9 − 6.55·log(hte))·log(d)
fc in MHz, hte and hre in meters, d in km. Remember to apply the correct a(hre) correction factor for city size.

Erlang B Traffic Formula

Traffic per user: A_u = λ × H
Total traffic: A = A_u × U
GOS (Erlang B): use table in paper

Market penetration = Users supported / Total population
Erlang B table is always attached to the exam. Find N (channels) from cells×channels/cell, look up A for given blocking %.

Coherence Bandwidth & Delay Spread

σ_τ = √(Στ²P(τ)/ΣP(τ) − [ΣτP(τ)/ΣP(τ)]²)

Bc ≈ 1/(50·σ_τ) [90% coherence]
Bc ≈ 1/(5·σ_τ) [50% coherence]
If Bc >> Bs (signal BW) → flat fading. If Bc < Bs → frequency selective fading. Compare with system BW.
Active Recall

How to test yourself for this exam

EX 751 tests both numericals and conceptual understanding. These testing methods are designed specifically for this subject.

📐

Solve numericals without looking

Pick any calculation from past papers. Work through it with formula sheet only — no worked examples. Then verify your method.

  • Priority: Erlang B, FDMA calc, Hata model, free-space propagation
  • Write out each step explicitly — marks are awarded for method
  • Check final units match expected output
  • Time yourself: 10–15 min per 5-mark problem
📊

Draw block diagrams from memory

Most diagram questions carry 6–8 marks. You must draw them correctly from memory. No partial credit for incomplete diagrams.

  • GSM architecture with all block names + interfaces
  • OFDM transmitter and receiver block diagrams
  • RAKE receiver (M-branch) block diagram
  • CDMA encoding/decoding process diagram
📋

Write comparison tables

Many questions ask you to "compare" two systems. A well-structured table earns full marks. Practice writing these quickly.

  • FDMA vs TDMA vs CDMA — 3 columns minimum
  • 1G vs 2G vs 3G vs 4G vs 5G — key parameters
  • Rayleigh vs Rician fading distributions
  • GSM vs CDMA — architecture and standards
🗣️

Explain concepts out loud (Feynman method)

Pick any theoretical concept and explain it to an imaginary student. If you stumble, that's exactly what you need to re-study.

  • "Why does OFDM need a cyclic prefix?"
  • "How does CDMA separate users in the same frequency?"
  • "What exactly happens during a handoff in GSM?"
  • "Why is the coherence bandwidth related to delay spread?"
📄

Full paper simulation

Do complete past papers strictly timed. 80 marks in 3 hours = 2.25 minutes per mark. Know your pacing before exam day.

  • Start with questions you know best (build confidence)
  • Allocate 15–20 min per 8-mark question
  • Never skip a question entirely — always write something
  • Start with 2079, then 2078, then 2077 paper
❌

Build your error log

Every mistake is information. Keep a running list of every question you got wrong or incomplete. Review it daily in the final week.

  • Write the topic, the question, and what you got wrong
  • Note whether it was a concept gap or a careless error
  • Re-attempt the same question 48 hours later
  • Track your confidence per chapter (1–5 scale)
Exam Strategy

TU IOE exam-specific tips

Based on how TU IOE exams are marked and structured. These apply specifically to EX 751.

1

Always show your working

TU IOE examiners give marks for method, not just the final answer. Even if your answer is wrong, correct working can earn 3 out of 5 marks. Never skip steps.

2

The FDMA problem is a gift

The FDMA channel count numerical (12.8 MHz / 30 kHz) has appeared in almost every exam since 2073. It is a free 4 marks if you memorise the formula and substitution.

3

Attempt ALL questions

The paper says "Attempt All Questions." There are no optional questions. Even for questions you're unsure about, write what you know — partial credit adds up to passing.

4

The Erlang B table is given

The Erlang B / Erlang C table is always attached to the exam paper. You don't need to memorise the formula — just know how to read the table and identify the correct row and column.

5

Label ALL diagrams

Block diagrams without labels get partial or no credit. Every block, signal line, and interface in GSM architecture, OFDM, RAKE receiver must be labeled clearly.

6

Short notes format

Short notes (2×3 or 2×4 marks) typically need: definition, key features, block diagram or equation, application. 4 bullets per note = safe marks.

7

GSM and Ch2 carry the most marks

In every paper examined, Ch2 (cellular concepts) and Ch8 (GSM) together carry 20–26 marks. If you master only two chapters, make it these. They are also predictable.

8

State assumptions clearly

The paper says "Assume suitable data if necessary." If a question is missing a parameter (e.g. path loss exponent), state your assumption and proceed. Examiners expect this.
